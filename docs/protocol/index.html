Aviora — Protocol Layer

JSON-over-TCP between the device and the central server: AppTcpConnManager (push/pull sockets; received bytes are passed to the protocol via IncomingMsngCb_t at appTcpConnManagerStart), AppMeterOperations (meter and directive jobs), AppProtocolZD (message flow and state machine), and a Python test server for development.

JSON / TCP Push + Pull Ident / Alive Readout / Load Profile

Push vs pull

Channel Who initiates? Typical messages
Push Device opens TCP to the server ident, alive, readout/load profile data packets
Pull Device listens locally; server connects in log, setting, fwUpdate, requests, directive CRUD
