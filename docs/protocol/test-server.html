AVIORA Protocol Test Server

Desktop Test Server application listens on the push port where the device connects, reads pullIP / pullPort from the first ident message (or uses the TCP source IP when needed) and sends pull commands to that address.

How to run

cd Application\AppZMeterGw\Services\Protocol\Test_Server

python ProtocolZD_TestServer.py                 # default push port: 8722
python ProtocolZD_TestServer.py 9000            # custom port

python ProtocolOrionTLV_TestServer.py            # default push port: 8723
python ProtocolOrionTLV_TestServer.py 9000      # custom port

After the window opens, click Start to begin listening. The device must be configured to reach this machine’s IP and the same port.

Test Server — screenshots (wireframes)

Features

  • Push listener — Single client; logs ident, alive, readout/load profile data.
  • Auto replies — Optional register: true for ident, ack for alive and end-of-data (can be disabled).
  • Pull client — Buttons stay disabled until ident; then connects to pullIP:pullPort and sends JSON.
  • Multi-packet — For log and directive list, reads until packetStream is false.
  • File log — zd_server_log_YYYYMMDD_HHMMSS.txt is created automatically.
  • Loopback fix — If the device reports 127.0.0.1, the server can use the real TCP source IP as the pull target.

Sample scenarios

Scenario A — First registration

  1. Run python ProtocolZD_TestServer.py 8722, then click Start.
  2. Configure the gateway push target as this_PC_IP:8722.
  3. Device sends ident; JSON appears in the log; optional auto register: true is sent back.
  4. Pull IP/port update; try Log request or Add meter.

Scenario B — Readout

  1. First use Add meter to register a meter.
  2. Use Readout request and enter directive and serial.
  3. You should see ack on pull, then packetized readout on push.

Scenario C — Change server address

  1. Use Server settings to send a new IP/port; the device restarts TCP (next connection uses the new address on real hardware).
