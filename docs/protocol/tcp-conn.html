AppTcpConnManager

AppTcpConnManager.c manages the gateway’s push connection to the central server and the local pull TCP server in a single OS task.

Role

  • Push (PUSH_TCP_SOCK_NAME = "push"): The device connects to the configured server IP/port; ident, alive, and outbound data use this channel. Server responses are read on the same socket.
  • Pull (PULL_TCP_SOCK_NAME = "pull"): The device listens on a local port; the head-end (or test tool) connects and sends commands.
  • Incoming data is delivered through the IncomingMsngCb_t callback passed to appTcpConnManagerStart(..., incomingMsngCb). The ZMeter JSON stack uses appProtocolZDPutIncomingMessage from AppProtocolZD.c; another module can supply its own handler.

Lifecycle flow

  B --> C{taskKeepDisconnect?}
  C -->|optional| D[Close pull/push]
  C -->|no| E[taskPullSocCreat → createPullSocket]
  E --> F[taskConnectPush → connectPushSocket]
  B --> G[select: pull accept / client read]
  B --> H[select: push read / write ready]
  G -->|data| I["incomingMsngCb pull"]
  H -->|data| J["incomingMsngCb push"]

Key APIs

FunctionDescription
IncomingMsngCb_tType of incomingMsngCb: void (*)(const char *channel, const char *data, unsigned int dataLength). Channel is "push" or "pull".
appTcpConnManagerStart(ip, port, pullPort, incomingMsngCb)Stores server address, local pull port, and the incoming-message callback; starts the TCP thread.
appTcpConnManagerRequestConnectCreates the pull socket; allows push to connect.
appTcpConnManagerRequestPushConnectQueues a push connection request.
appTcpConnManagerSend(ch, data, len)Send on "push" or "pull" channel.
appTcpConnManagerIsConnectedPushReturns whether push is connected and handshake completed.
